# Manual do Usuário — PainelLicita
## Prefeitura Municipal de Osasco — Secretaria de Compras

> **Para quem é este manual?**
> Este documento é destinado a gestores, pregoeiros, diretores e demais colaboradores que utilizam o PainelLicita no dia a dia. Não é necessário nenhum conhecimento técnico para lê-lo.

---

## O que é o PainelLicita?

O **PainelLicita** é o painel de monitoramento de pregões eletrônicos da Secretaria de Compras da Prefeitura Municipal de Osasco. Ele centraliza, em uma única tela, o status atualizado de todos os processos de licitação em andamento e encerrados — eliminando a necessidade de consultas manuais em planilhas ou ligações para saber a situação de cada pregão.

### Para que serve?

| Necessidade | Como o PainelLicita resolve |
|---|---|
| Saber quais pregões estão acontecendo agora | Painel principal exibe os processos ativos em tempo real |
| Verificar se um pregão foi suspenso | Status **SUSPENSO** aparece com destaque em vermelho |
| Consultar o histórico de licitações encerradas | Aba **Encerrados** reúne todos os processos finalizados |
| Acompanhar sem precisar acessar planilhas | Os dados são atualizados automaticamente a cada 10 segundos |

---

## Como acessar o sistema

O PainelLicita é acessado diretamente pelo navegador (Chrome, Edge, Firefox). Não é necessário instalar nenhum programa, criar conta ou fazer login. Basta abrir o endereço fornecido pela equipe de TI.

O painel pode ser exibido em:
- Computadores e notebooks
- TVs e monitores de sala de operações (modo tela cheia)
- Tablets e celulares (a tela se adapta automaticamente)

---

## Navegando pelo sistema

O sistema possui **3 telas principais**, acessíveis pelo menu superior:

```
[ Início / Painel ]   [ Ver Todos ]   [ Encerrados ]
```

---

## Tela 1 — Painel Principal (Início)

Esta é a tela inicial do sistema. Ela mostra **todos os pregões ativos** do momento, com atualização automática a cada **10 segundos**.

### Seção "Pregões do Dia"

No topo da tela, aparecem até **4 pregões em destaque**. São priorizados automaticamente:

- Pregões com sessão **Em Andamento** no momento
- Pregões com data de abertura marcada para **hoje**
- Pregões marcados como urgentes ou prioritários pela equipe

> Este destaque funciona como um "alerta visual" para que a equipe saiba imediatamente quais processos exigem atenção agora.

### Seção "Painel de Monitoramento Geral"

Logo abaixo, uma grade com os **demais pregões ativos** é exibida (até 15 itens visíveis). Cada card mostra:

| Informação | O que representa |
|---|---|
| **Número do PE** | Identificação do Pregão Eletrônico (ex: PE 042) |
| **Status** | Situação atual do processo (veja tabela de status abaixo) |
| **Pregoeiro Responsável** | Nome do servidor responsável por conduzir o pregão |
| **Objeto** | Descrição resumida do que está sendo licitado |
| **Data de Abertura** | Data em que a sessão do pregão foi ou será aberta |
| **Sub-status** | Informação adicional quando necessário (ex: "Aguardando recurso") |

Se houver mais de 15 pregões ativos, um link **"Ver Todos"** aparece para acessar a lista completa.

### Rodapé — Totalizadores por Status

Na parte inferior da tela, um painel de resumo exibe a contagem de processos por situação, atualizada em tempo real:

| Indicador | Cor | Significado |
|---|---|---|
| **Em Andamento** | Verde | Sessões de pregão abertas e ocorrendo agora |
| **Suspenso** | Vermelho | Processos interrompidos temporariamente |
| **Decisão / Análise** | Roxo | Processos aguardando análise técnica ou decisão jurídica |
| **Aguardando** | Cinza | Processos agendados, aguardando a data de abertura |
| **Aguardando Edital** | Amarelo | Processos aguardando a publicação do edital |

---

## Tela 2 — Ver Todos

Quando o número de pregões ativos ultrapassa os 15 exibidos no painel principal, esta tela mostra os **itens excedentes** no mesmo formato de cards.

Use esta tela quando precisar consultar um processo que não aparece na grade principal. As informações exibidas são idênticas às do Painel Principal.

---

## Tela 3 — Encerrados

Esta tela funciona como o **arquivo histórico** de todos os pregões que já foram finalizados. Os dados são atualizados a cada **60 segundos**.

### O que é exibido em cada card:

| Informação | O que representa |
|---|---|
| **PE** | Número do Pregão Eletrônico |
| **Pregoeiro** | Responsável pelo processo |
| **Objeto** | Descrição do item licitado |
| **Data de Abertura** | Quando a sessão foi aberta |
| **Data de Encerramento** | Quando o processo foi finalizado |
| **Data de Homologação** | Quando o resultado foi homologado pela autoridade competente |
| **Observação** | Notas relevantes sobre o processo (ex: "Deserto", "Fracassado", observações jurídicas) |

> Todos os cards exibem o status **ENCERRADO** em cinza, indicando que o processo não está mais ativo.

## Entendendo os Status dos Pregões

Os status são a informação mais importante do painel. Veja o que cada um significa:

### Status Ativos

| Status | Cor no painel | Significado |
|---|---|---|
| **EM ANDAMENTO** | Azul / Destaque laranja | A sessão do pregão está aberta e acontecendo agora. O pregoeiro está conduzindo as rodadas de lances ou análise de propostas. |
| **AGUARDANDO** | Cinza | O pregão está cadastrado, mas ainda não chegou a data de abertura da sessão, ou está aguardando alguma providência interna. |
| **AGUARDANDO EDITAL** | Amarelo | O processo existe, mas o edital de licitação ainda não foi publicado no Diário Oficial ou no sistema eletrônico. |
| **EM ANÁLISE** | Roxo | O pregão está em fase de análise — pode ser análise de habilitação, análise de documentos, análise de recursos, laudo técnico, etc. |
| **DECISÃO** | Roxo | O processo está aguardando uma decisão formal — geralmente decisão sobre recurso, impugnação ou ato do gestor/jurídico. |
| **SUSPENSO** | Vermelho / Destaque vermelho | O pregão foi **interrompido** temporariamente. Pode ser por impugnação, decisão judicial, necessidade de correção no edital ou outro motivo. Requer atenção imediata. |

### Status de Histórico

| Status | Cor no painel | Significado |
|---|---|---|
| **ENCERRADO** | Cinza | O processo foi concluído. Aparece apenas na tela **Encerrados**. |

## Quem usa o sistema e como

### Gestor / Diretor de Compras

**Como usa o painel:**
- Abre o painel no início do dia para ver a situação geral de todos os pregões
- Acompanha os totalizadores no rodapé para ter visão macro: quantos estão em andamento, suspensos, aguardando
- Identifica rapidamente pregões **SUSPENSOS** (destacados em vermelho) que exigem providências
- Usa a tela **Encerrados** para consultar o histórico e verificar datas de homologação

**O que o painel oferece ao gestor:**
> Visão gerencial completa do portfólio de licitações em uma única tela, sem precisar consultar planilhas individuais ou ligar para pregoeiros.

---

### Pregoeiro

**Como usa o painel:**
- Verifica se seu pregão está corretamente exibido com o status atualizado
- Confirma com a equipe de suporte quando um status precisa ser alterado (ex: de AGUARDANDO para EM ANDAMENTO)
- Usa como referência rápida para ver a agenda do dia na seção "Pregões do Dia"

**Como o pregoeiro atualiza o status do seu pregão:**
> O pregoeiro comunica à equipe responsável pela planilha (Google Sheets) para atualizar o status. O painel reflete a mudança automaticamente em até 10 segundos.

---

### Equipe de Apoio / Analistas

**Como usam o painel:**
- Monitoram a tela em tempo real durante o dia para identificar mudanças de status
- Consultam a tela **Encerrados** para buscar informações de pregões anteriores (datas, observações, pregoeiro responsável)
- Verificam o contador de processos "Em Análise" para dimensionar a fila de trabalho

---

### Equipe que mantém os dados (Planilha Google Sheets)

**Responsabilidade:**
A equipe responsável pela manutenção alimenta uma planilha no Google Sheets, que é a fonte de dados do painel. Para atualizar o painel basta editar a planilha — as informações aparecem automaticamente na tela em até 10 segundos.

**Estrutura da planilha (aba principal — Pregões Ativos):**

| Coluna | Dado | Exemplo |
|---|---|---|
| A | Número do PE | `PE 042` |
| B | Nome do Pregoeiro | `João da Silva` |
| C | Objeto da licitação | `Aquisição de material de limpeza` |
| D | Data de abertura | `15/03/2026` |
| E | Status | `EM ANDAMENTO` |

**Estrutura da planilha (aba — Encerrados 2026):**

| Coluna | Dado | Exemplo |
|---|---|---|
| A | Número do PE | `PE 010` |
| B | Nome do Pregoeiro | `Maria Souza` |
| C | Objeto | `Serviços de vigilância` |
| D | Data de Abertura | `10/01/2026` |
| E | Data de Encerramento | `20/01/2026` |
| F | Data de Homologação | `25/01/2026` |
| G | Observação | `Homologado parcialmente` |

**Palavras-chave reconhecidas automaticamente pelo sistema:**

O sistema lê o campo de status e interpreta automaticamente. Use as palavras abaixo no campo de status da planilha:

| Palavra-chave na planilha | Status exibido no painel |
|---|---|
| `ANDAMENTO` | Em Andamento (destaque azul/laranja) |
| `SUSPENSO` | Suspenso (destaque vermelho) |
| `DECISÃO` | Decisão (roxo) |
| `ANÁLISE` ou `LAUDO` | Em Análise (roxo) |
| `EDITAL` | Aguardando Edital (amarelo) |
| *(em branco ou outro texto)* | Aguardando (cinza) |

## Perguntas Frequentes

**O painel não está atualizando. O que fazer?**
> Verifique se há conexão com a internet. O painel depende de acesso ao Google Sheets para buscar os dados. Se a conexão estiver ativa e o problema persistir, acione a equipe de TI.

**Atualizei a planilha mas o painel ainda mostra o status antigo. Por quê?**
> O painel atualiza a cada 10 segundos. Aguarde alguns instantes após salvar a planilha. Se após 30 segundos a informação não mudar, verifique se a planilha está publicada corretamente (configuração feita pela TI).

**Um pregão não aparece no painel. O que pode ter acontecido?**
> Verifique se a linha correspondente foi preenchida corretamente na planilha, especialmente o número do PE (coluna A). Linhas sem o número do PE são ignoradas automaticamente pelo sistema.

**Como consultar pregões de anos anteriores?**
> Atualmente o painel exibe apenas os pregões da aba **Encerrados 2026**. Para anos anteriores, consulte diretamente a planilha do Google Sheets com a equipe responsável.

**O sistema funciona no celular?**
> Sim. O painel se adapta ao tamanho da tela automaticamente. Em telas pequenas, os cards se reorganizam em uma coluna para facilitar a leitura.

**Quem pode ver o painel?**
> Qualquer pessoa que tenha o endereço (link) do painel pode acessá-lo. Não há senha ou login. Recomenda-se que o link seja compartilhado apenas internamente.
